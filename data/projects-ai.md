# AI / LLM проекты

## job-hunter — авто-отклики на вакансии с LLM-отбором
- Задача: искать вакансии (Python/AI, remote), семантически отбирать релевантные
  (в т.ч. «компания использует ИИ» без явного упоминания), генерировать персональное
  сопроводительное письмо и автоматически откликаться; вести переписку с рекрутёрами.
- Стек: Python, hh.ru API (OAuth), SQLite (state/dedup), Playwright (отклики и чаты),
  Windows Task Scheduler; LLM — Codex CLI (gpt-5.5) основной + claude -p фоллбэк через
  единый модуль llm.py; Telegram-бот для ленты; pytest + TDD.
- Архитектура: модульный пайплайн fetch → filter → classify → letter → apply →
  state + notify. Источники за интерфейсом Source. LLM-классификация (правила +
  семантика). Чат-модуль: гибридная маршрутизация — простые вопросы агент отвечает
  сам, чувствительные (деньги/переезд/ПД/тест) уходят черновиком пользователю в
  Telegram. Дедуп по SQLite (last_msg_id).
- Роль: единственный разработчик — дизайн, пайплайн, промпты, browser-автоматизация,
  состояние, деплой.
- Результат: автономный цикл «нашёл → отобрал → письмо → откликнулся → ответил».

## notion — классификатор сообщений в задачи
- Задача: из потока Telegram-чатов выделять реальные задачи, отсекать шум.
- Стек: Python, AsyncIO, Telethon, Claude Haiku через claude -p, Docker на VPS.
- Архитектура: Telethon fetch → filter → rules + Haiku classify → task_writer.
  Двухступенчатая классификация: дешёвые правила + LLM на «серую зону». Дедуп,
  группировка по датам.
- Роль: разработчик пайплайна, промптов классификатора, правил маршрутизации.
- Результат: входящие чаты превращаются в чистый список задач без ручной разборки.

## tracking-movements — LLM-парсинг заказов в перемещения товара
- Задача: из неструктурированных комментариев заказов нескольких магазинов извлекать
  структурированные перемещения товара между точками + трекинг доставки.
- Стек: Python 3.14, AsyncIO, OpenAI API (ChatGPT mini), e-commerce API магазинов, pytest,
  Docker + Kubernetes (Helm), GitLab CI/CD.
- Архитектура: API магазинов → LLM-извлечение структуры → нормализация (алиасы точек,
  сопоставление товаров по полю model) → хранение перемещений как файлов → нотификации.
- Роль: разработчик интеграций, LLM-извлечения, нормализации, деплоя в K8s.
- Результат: свободный текст комментариев → структурированные перемещения; трекинг
  почтовых отправлений.

## autootvetchikchatgpt / whatsapp — ChatGPT авто-ответчик
- Задача: автоматически отвечать клиентам в Telegram/WhatsApp на основе промптов и
  контекста аренды/товаров.
- Стек: Python, OpenAI API, интеграции мессенджеров.
- Архитектура: входящее сообщение → подбор промпта/контекста → генерация LLM →
  отправка в канал.
- Роль: разработчик авто-ответчика и системы промптов.
- Результат: типовые обращения клиентов закрываются без оператора.

## rag-service — RAG по документации опыта (этот проект)
- Задача: отвечать на вопросы о профессиональном опыте кандидата по корпусу
  документации (проекты, навыки, стек) с цитированием источников.
- Стек: Python, Qdrant (vector DB), fastembed (локальные эмбеддинги, multilingual-e5),
  FastAPI, Codex CLI / claude -p для генерации, pytest.
- Архитектура: ingest (md/pdf → чанкинг → эмбеддинги → Qdrant) → retrieval (top-k по
  косинусу) → подмешивание контекста в промпт → генерация ответа с источниками.
- Результат: рабочий RAG-движок с CLI и HTTP API, демонстрируемый на собственном
  портфолио; есть eval-харнесс (hit@k, latency).
